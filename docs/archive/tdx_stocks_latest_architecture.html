tdx-stocks Latest Architecture

现在的主架构把命令行入口收束为领域路由：`sync` 是宏命令，`data` / `audit` / `query` 是主分组；写操作通过 `Database/.lock` 串行化，`--dry-run` 走纯只读计划，旧命令仍保留为兼容别名。

Control Plane

  • • `sync` decides whether update + rebuild are needed.
  • • `sync` refreshes caches; `status` audits them.
  • • `--dry-run` and query commands stay lock-free.

Write Path

  • • `build` / `rebuild` stay offline and reuse local cache only.
  • • Generates `adj_daily`, `hfq_daily`, `factors`.
  • • `Database/.lock` serializes all write paths.

Read Path

  • • `latest.json` resolves a single immutable version.
  • • `query stock` selects raw / qfq / hfq bases.
  • • `status`, `tables`, `schema`, `head`, `sql`, `export`, `actions-status` stay offline.
